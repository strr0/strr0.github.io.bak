springcloud-day3

SpringCloud - Day3

Hystrix的客户端弹性模式

  1. 断路器模式
    杀死运行缓慢和退化的系统调用，调用快速失败，防止资源枯竭
  2. 回退模式
    调用失败或调用的断路器失败时定义的替代代码路径
  3. 舱壁模式
    调用隔离在自己的线程池，防止吞噬所有资源

项目搭建

  1. 新建maven项目spmia-chapter5
    添加maven依赖（cloud2020已移除hystrix）
  2. 在spima-chapter5下创建子模块confsvr
    添加maven依赖 bootstrap.yml配置（encrypt一定要配置在此，否则会使加密失效） application.yml配置
  3. 在spima-chapter3下创建子模块licensing-service
    添加maven依赖（Hoxton.SR5版本无需添加bootstrap依赖）
  4. 在licensing-service的Application中添加注解 @EnableCircuitBreaker 开始断路器模式
  • 断路器模式
    在指定方法上添加hystrix注解，断路器默认超时为1s，超过则调用失败
  • 回退模式
    在指定方法上添加hystrix注解，并添加回退方法
  • 舱壁模式（在自己的线程池中隔离远程资源调用，防止性能不好的服务导致容器崩溃）
    在指定方法上添加hystrix注解

  1. 在spmia-chapter4下创建子模块eurekasvr（eureka服务端）
    添加maven依赖

    eureka服务端配置（不在eureka注册和本地缓存）

    在启动类添加注解 @EnableEurekaServer 开启eureka服务
    项目启动后访问 http://localhost:8761/eureka/apps 可查看eureka服务

  2. 在spmia-chapter4下创建子模块organization-service（eureka客户端）
    添加依赖

    organization-service配置（defaultZone为eureka服务地址）

  3. 在spmia-chapter4下创建子模块licensin-service（eureka客户端）
    添加依赖

    licensing-service配置（eureka客户端）

  4. 发现eureka服务的三种方式

  • DiscoveryClient
    在当前项目（licensing-service）的启动类添加注解 @EnableDiscoveryClient 开启服务发现
    使用DiscoveryClient的getInstances(serviceId)获取服务实例，从实例获取uri（uri + 接口）访问
  • RestTemplate
    在RestTemplate的bean上添加注解 @LoadBalanced 告诉SpringCloud创建一个支持Ribbon的bean
    使用RestTemplate访问serviceId（http://serviceId + 接口）
  • FeighClient
    在当前项目（licensing-service）的启动类上添加注解 @EnableFeignClients 开启feign
    创建feignclient接口，并添加注解 @FeignClient(serviceId)
    在feignclient接口下添加方法，并添加注解 @RequestMapping

SpringCloud - Day1

配置中心配置加密

  1. 下载无长度限制的jce_policy-8
  2. 解压并替换jre/secret下的同名文件（记得先备份）
  3. 在环境变量中添加ENCRYPY_KEY变量，SpringCloudConfig检测到变量，会自动添加两个新端口（/encrypt和/decrypt)，用于加密和解密，此变量也可直接写到application.yml中（不推荐）。
  4. 使用/encrypt端口加密字段并写入配置（xxxxxx为加密后的）
  5. 在服务端配置不解密
  6. 在客户端添加依赖
  7. 在客户端添加ENCRYPY_KEY变量，用于解密，也可以写到application.yml中（不推荐）。

springcloud-day1

SpringCloud - Day1

大致构成

  1. 服务装配：打包和部署微服务
  2. 服务引导：管理微服务的配置
  3. 服务注册与发现：客户端如何与微服务通信
  4. 监控微服务健康状况

项目搭建

  1. 新建maven项目spmia-chapter3（此项目为父目录，无需编写实际业务代码）
    添加maven依赖 此步实际没有添加依赖，而是为子模块指定springboot、springcloud版本
  2. 在spima-chapter3下创建子模块confsvr（配置中心）
    添加maven依赖，这两个依赖为配置中心所需 在resources下创建application.yml，search-locations就是配置所在的路径，多个可以用’,’隔开 创建入口ConfigServerApplication（EnableConfigServer注解开启配置中心） 运行后访问http://localhost:8888/licensingservice/default即可看到配置中心所提供的配置
  3. 在spima-chapter3下创建子模块licensing-service（客户端）
    添加maven依赖 在resources下创建bootstrap.yml（从localhost:8888读取配置） 创建入口Application（RefreshScope注解用于刷新自定义配置，访问接口refresh刷新）
  4. 踩坑
    在bootstrap添加了配置，但是读取不到配置中心的配置
    尝试将配置复制到application中还是不行
    几番尝试后发现，从配置中心读取配置需要在bootstrap中进行配置，但是bootstrap的配置无效
    解决方案参考https://blog.csdn.net/qq_28336351/article/details/113730679
    因为在springcloud2020版本之后对配置做了一些修改导致boostrap失效，可添加依赖spring-cloud-starter-bootstrap开启boostrap配置

hexo搭建个人博客

准备

  1. nodejs
  2. hexo
  3. git、github

环境搭建

  1. nodejs安装（hexo是基于nodejs的，已安装请跳过）
  • 从官网下载nodejs（推荐选择LTS版本）并安装
  • 配置nodejs 全局安装路径和缓存路径（路径例如E:\Users\xxx\node\node_global）
  • 将全局安装路径添加到环境变量（全局使用npm安装的组件如hexo）

  1. hexo安装

  2. hexo基本使用
    博客存放目录 cd '路径'
    博客初始化 hexo init
    生成博客页面 hexo g
    启动预览 hexo s ，可在http://localhost:4000查看预览

  3. git安装

  • 从官网下载git并安装
  • 配置git
  • 创建github page页面，即page仓库（例如xxx.github.io）
  1. hexo配置
  • 安装hexo-deployer-git
  • 修改 _config.yml 文件末尾的 Deployment 部分
  • 运行命令 hexo d 即可发布到github

博客使用

  1. 进入博客所在目录 cd '目录'
  2. 使用命令 hexo new '博文标题' 创建博客，博客目录的source下会出现md文件
  3. 在md文件中即可编写博文内容
  4. 使用命令 hexo g 生成页面，使用命令 hexo d 发布到github
参考

https://zhuanlan.zhihu.com/p/26625249
